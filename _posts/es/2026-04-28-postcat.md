Jekyll ek static site generator hai jo Markdown files ko sundar HTML pages mein badal deta hai. GitHub Pages isko seedha support karta hai — matlab aapko koi hosting kharidni nahi padti.

## Jekyll Kyun Use Karein?

Bahut saare blogging platforms available hain — WordPress, Ghost, Medium — lekin Jekyll ke apne fayde hain:

- **Free hosting** — GitHub Pages pe bilkul free deploy hota hai
- **Speed** — static HTML hone ki wajah se page load bahut fast hota hai
- **Version control** — saare posts Git mein tracked rehte hain
- **No database** — sirf plain text files, kuch nahi toota

## Setup Kaise Karein

Pehle apne machine pe Ruby aur Jekyll install karo:

```bash
gem install bundler jekyll
jekyll new mera-blog
cd mera-blog
bundle exec jekyll serve
```

Ab browser mein `http://localhost:4000` kholo — tera blog live ho gaya locally.

## Folder Structure

```
mera-blog/
├── _layouts/
│   └── blog.html        # post ka template
├── _includes/
│   └── header.html      # reusable parts
├── _posts/
│   └── 2024-01-15-mera-pehla-post.md
├── category.html        # category archive page
├── tag.html             # tag archive page
└── _config.yml
```

## Post Likhna

Har post `_posts/` folder mein `YYYY-MM-DD-title.md` format mein hoti hai. Front matter kuch aisa dikhta hai:

```yaml
---
layout: blog
title: "Mera Post Title"
date: 2024-01-15
categories: [tech, tutorial]
tags: [jekyll, github-pages]
---
```

Dhyan rakho — `categories` likhna hai, `category` nahi. Jekyll sirf `categories` aur `tags` ko `post.categories` / `post.tags` ke through access karta hai.

## Category aur Tag Links

Post ke andar category aur tag links kuch is tarah daal sakte ho:

```html
{% for cat in post.categories %}
  <a href="/blog/category/?c={{ cat | slugify }}">{{ cat }}</a>
{% endfor %}
```

Yeh link reader ko seedha us category ke saare posts pe le jaayega — bina koi alag folder banaye.

## GitHub Pages Pe Deploy

1. GitHub pe naya repository banao
2. Repository ka naam `username.github.io` rakho
3. Apna Jekyll blog push karo

```bash
git init
git add .
git commit -m "pehla commit"
git remote add origin https://github.com/username/username.github.io.git
git push -u origin main
```

Kuch minute baad tera blog `https://username.github.io` pe live ho jaayega.

## Conclusion

Jekyll simple hai, fast hai, aur GitHub Pages ke saath perfectly kaam karta hai. Ek baar setup ho jaaye toh bas Markdown likhte jao — baki sab Jekyll khud sambhal leta hai.

Agar categories aur tags ka universal layout chahiye — bina alag folder banaye — toh `category.html` aur `tag.html` wali approach use karo jo hum pehle bana chuke hain.
